# =====================================================================
# DEPENDENCIAS DE SISTEMA EN LINUX (no son paquetes de pip — instalar
# con el gestor de paquetes de la distro antes de correr el asistente)
#
#   Debian/Ubuntu:  sudo apt install wmctrl xdotool playerctl pulseaudio-utils
#   Fedora:         sudo dnf install wmctrl xdotool playerctl pulseaudio-utils
#   Arch:           sudo pacman -S wmctrl xdotool playerctl libpulse
#
# Sin estas, el asistente sigue arrancando y funcionando (voz,
# recordatorios, macros, alias, no molestar, abrir/cerrar apps,
# comandos de sistema) — pero cada función específica que dependa de
# la utilidad faltante avisa con claridad en consola/log en vez de
# fallar en silencio (ver _wmctrl_disponible/_xdotool_disponible en
# ventanas_linux.py, _playerctl_disponible/_pactl_disponible en
# media_control_linux.py).
#
# También se asume systemd (systemctl/loginctl) para apagar/reiniciar/
# suspender/bloquear la PC y para el modo híbrido de energía — es el
# init system por defecto en Ubuntu, Fedora, Debian, Arch y la
# inmensa mayoría de distros de escritorio actuales. En una distro
# sin systemd (ej. Void Linux con runit) esos comandos puntuales no
# van a funcionar; el resto del asistente no se ve afectado.
# =====================================================================
